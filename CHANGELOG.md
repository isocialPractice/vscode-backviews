# Changelog

All notable changes to this project will be documented in this file. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres to
[Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.0.1-alpha] - 2026-07-16

### Fixed

- Player collision glitch at open doorway jamb surfaces: resolved rapid back-and-forth motion
  ("punched" or shaken feeling) when pressing into the extruded jamb face of an open doorway.
  The collision resolver's wall-clamp was unbounded, so a player already standing legally
  close to a jamb (having entered through the opening) would be snapped outward by a full
  PLAYER_RADIUS every blocked frame, then allowed back the next, causing continuous
  oscillation. Fixed by adding a `withinStep` bounds check that restricts the clamp target
  to the span actually travelled in the current frame. All other wall collision is unchanged.

## [0.0.0-alpha] - 2026-07-15

### Added

- Browser demo: standalone `index.html` that runs BackViews directly in the browser without 
  VSCode, simulating the VSCode API for settings persistence via localStorage
- GitHub Actions workflow for automatic deployment to GitHub Pages on push to main branch
- Browser demo documentation in README with local testing instructions
- GitHub Pages setup guide at `.github/GITHUB_PAGES_SETUP.md`

### Fixed

- GitHub Actions workflow updated to Node 20 (avoiding Node 18 deprecation warnings)
- Removed npm caching from workflow to prevent "unable to cache dependencies" errors

### Added (Core Features)

- First-person backrooms explorer in a VSCode webview panel, generated by the
  [cmd-backedges](https://github.com/isocialPractice/cmd-backedges) infinite maze engine
  (walls extruded from edge lines, ceiling height from extrusion depth, props from cell
  features).
- Backrooms material set painted procedurally into one texture atlas: yellow wallpaper walls,
  drop-tile office ceilings, backlit plastic light panels hung just below the ceiling, and
  worn fiber carpet.
- Photo materials: `materials/wallpaper.jpg`, `materials/ceiling.jpg`, and
  `materials/carpet.jpg` are tiled over the wall, ceiling, and floor atlas tiles at startup,
  with per-file fallback to the procedural textures. Swapping the files re-skins the game.
- Movement on WASD and arrow keys, with strafing, turning, Shift to hurry, and optional
  pointer-lock mouse look. Turn, strafe, and forward/back can each be inverted.
- In-game menu (`M` / `Esc`) with Resume, Relocate, Settings, Help, and live world stats.
- Settings in VSCode config and in-game: seed, walk speed, render distance, camera shake,
  film grain, camcorder HUD, furniture, wallpaper shifts, mouse look, and control inversion.
- Wall material presets selectable from a dropdown (classic wallpaper mix, drywall, ceramic
  tile, concrete, wood paneling) with hue-shift and brightness sliders.
- The monster: dormant until a randomized time inside a configurable spawn window, then it
  stalks the player through the same open passages the collision system uses. Configurable
  on/off, speed, spawn window, and body plan (spider-like, human-like, cloud-like, or a random
  form each spawn). Getting caught cuts the tape to static and restarts from the origin cell.
- Random furniture: filing cabinets, desks, couches, box stacks, and columns from
  generator-placed features.
- Wallpaper zones: randomly shaped enclosed blob regions where wallpaper tint and pattern
  change. Off by default; enable with `backviews.wallpaperShifts`.
- Wall extrusion depth: walls render as slabs with inset faces, doorway openings get jamb
  caps on both sides, and passages punched through a wall line get a header with a soffit
  underside and sealed ends.
- Handheld camera shake and a grainy 1990-era camcorder overlay: grain, vignette, tracking
  tears, blinking REC dot, tape counter, battery pips, and burned-in timestamp.
- Ceiling light behavior: lit, dead, and flickering panels with baked local lighting and a
  global fluorescent flicker.
- Commands: `BackViews: Enter the Backrooms` and `BackViews: Relocate (New Seed)`.
- Logo (`media/logo.svg`), minimal icon (`media/icon.svg`, `media/icon.png`), and a
  dependency-free PNG rasterizer script (`npm run icons`).
- Docs: README (install, getting started, game play, API overview, architecture) and
  QUICKSTART.

### Fixed

- Movement is now camera-relative. The movement basis previously mirrored the view matrix, so
  walking forward tracked the compass instead of the camera and turned into a strafe once the
  camera rotated.
